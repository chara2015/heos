## 用户名规则

### 正版玩家用户名
- ✅ 只能包含字母、数字、下划线
- ✅ 长度：3-16个字符
- ✅ 格式：`^[a-zA-Z0-9_]{3,16}$`
- ✅ 示例：`Steve`, `Alex`, `Player123`, `My_Name`

### 离线玩家用户名
- ✅ 必须包含特殊符号：`+` `-` `.`
- ✅ 可以包含字母、数字、下划线
- ✅ 示例：
  - `Player+123`
  - `Test.User`
  - `User-Name`
  - `My_Player.2024`
  - `Steve+`
  - `Alex.Offline`

### 错误示例
| 用户名 | 类型 | 结果 |
|--------|------|------|
| `Steve` | 离线玩家 | ❌ 被踢出，提示添加符号 |
| `Player123` | 离线玩家 | ❌ 被踢出，提示添加符号 |
| `Test_User` | 离线玩家 | ❌ 被踢出，提示添加符号 |
| `Steve+123` | 正版玩家 | ⚠️ 可进入，但走离线流程 |

## 登录流程对比

### 正版玩家登录流程
```
1. 客户端连接服务器
2. 发送LoginHello数据包
3. 服务器检查用户名格式 ✅ 纯字母数字
4. 查询Mojang API ✅ 账号存在
5. 验证UUID ✅ 匹配
6. 继续Mojang正版验证
7. 验证成功，进入服务器
8. 自动设置为已认证
9. 可以正常游戏
```

### 离线玩家登录流程（正确用户名）
```
1. 客户端连接服务器
2. 发送LoginHello数据包
3. 服务器检查用户名格式 ✅ 包含特殊符号
4. 跳过Mojang验证
5. 使用离线UUID
6. 进入服务器
7. 设置为未认证状态
8. 显示登录/注册提示
9. 玩家执行 /login 或 /register
10. 验证成功，设置为已认证
11. 可以正常游戏
```

### 离线玩家登录流程（错误用户名）
```
1. 客户端连接服务器
2. 发送LoginHello数据包
3. 服务器检查用户名格式 ❌ 纯字母数字
4. 查询Mojang API
   - 情况A：无账号 → 显示提示，断开连接
   - 情况B：有账号但UUID不匹配 → 显示提示，断开连接
5. 玩家被踢出
6. 看到中英双语错误提示
```
